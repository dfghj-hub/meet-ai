# Manus 界面生成 Prompt（SellBoost 项目）

复制下面整段到 Manus（或 MCP 工具 `manus_generate_ui` 的 prompt 参数）使用。可按需替换「页面名称」和「页面描述」来生成不同页面。

---

## 通用项目上下文（可放在每次请求开头）

```
项目名称：SellBoost — AI 社交电商内容引擎。
产品定位：用户输入产品/服务描述，AI 一键生成小红书、抖音、微信、B站、快手等多平台带货文案与脚本。

技术栈（必须遵守）：
- Next.js 14 App Router，React 18，TypeScript，Tailwind CSS。
- 仅使用 Tailwind 类名 + 现有 design tokens，不引入新 UI 库。

设计规范（必须遵守）：
- 深色主题。主背景色 #0a0a12，卡片/表面用半透明白 + 毛玻璃（backdrop-blur）。
- 主色：橙到粉渐变（orange-500 → pink-500），用于 CTA、强调、焦点态；克制使用，避免廉价霓虹感。
- 字体：标题 Sora（font-heading），正文 Inter（font-body）。
- 圆角：卡片/按钮用 rounded-2xl 或 rounded-3xl，输入框 rounded-xl。
- 已有工具类：.glass-card、.btn-primary、.btn-ghost、.input-glass；新组件风格需与之一致。
- 信息层级清晰，留白充足，避免页面发灰发白；可适当使用环境光晕（blur 大圆）增强氛围。

输出要求：
1) 先给 IA（信息架构）+ 视觉规范摘要（间距、字号、颜色用法）。
2) 再给 React + Tailwind 的组件级代码（可直接放入 src/app/... 或 src/components/...）。
3) 标注 Desktop / Tablet / Mobile 的适配策略（Tailwind 断点：sm/md/lg）。
4) 所有面向用户的文案使用中文。
```

## Prompt 1：生成页 `/generate`（内容工作台）

```
【用上面的通用项目上下文】

请为 SellBoost 的「生成带货内容矩阵」页面（/generate）设计并输出一版美观、大气、强转化的界面。

页面功能要点：
- 页面头：主标题「生成带货内容矩阵」+ 副标题说明「输入产品信息，AI 生成多平台文案」。
- 左侧主区：产品描述（必填 textarea）、产品链接（可选 input）、分发平台多选（小红书/抖音/微信/B站/快手）、内容形式（图文种草 / 口播短视频 / 测评对比）、转化目标（品牌种草 / 提升互动 / 私域留资 / 直接成交）、内容角度偏好（可选输入）、品牌语音档案（可选勾选 + 下拉选择档案）。
- 主 CTA：「智能分析并生成策略矩阵」按钮，置底或显眼位置。
- 右侧边栏：生成历史列表（最近 5 条），每条展示产品名、平台标签、操作「复用参数」「查看结果」「删除」。
- 错误态：产品描述为空时提示，接口报错时在表单上方展示错误信息。
- 加载态：提交后全屏或主内容区展示 loading，文案「AI 正在深度拆解产品并构建内容策略矩阵...」。
- 结果态：同一页面切换为结果视图——产品分析摘要卡片、按平台 Tab 的文案变体列表、可验证结果说明、下载全部按钮、反馈组件。

请输出：IA + 视觉规范 + React+Tailwind 代码（可拆成多个组件），并说明响应式策略。
```

## Prompt 2：品牌语音页 `/brand`（多档案管理）

```
【用上面的通用项目上下文】

请为 SellBoost 的「品牌语音档案」页面（/brand）设计并输出一版清晰、专业的界面。

页面功能要点：
- 页面头：主标题「品牌语音档案」+ 副标题说明「管理多套品牌语感，生成内容时可选用」。
- 档案列表区：展示所有已保存档案（卡片或列表），每项显示档案名称、最后更新时间；支持选中（高亮当前生效档案）、删除、新建档案。
- 当前档案编辑区：选中某档案后，下方或右侧展示表单。表单字段包括：档案名称、品牌名、行业、产品类别、目标人群、品牌语调（专业/亲切/活泼等）、偏好词、禁用词、常用 CTA、竞品关键词、成功案例描述、语感强度滑块（0–10）。保存按钮。
- 新建档案：点击「新建档案」后清空表单，保存时写入列表并设为当前选中。
- 无档案时：引导用户创建第一个档案，表单仍可见。

请输出：IA + 视觉规范 + React+Tailwind 代码，并说明响应式策略。
```

## Prompt 3：首页 `/`（落地页）

```
【用上面的通用项目上下文】

请为 SellBoost 的首页（/）设计并输出一版现代、大气、高转化率的落地页。

页面结构要求：
- Hero：主标题 + 副标题 + 核心价值点（如「一次输入，多平台适配」「算法规则自动匹配」）+ 主 CTA 按钮（如「免费生成内容」跳转 /generate）+ 可选配图或动效区域。
- How it works：3–4 步说明（例如：输入产品信息 → 选择平台与目标 → AI 生成 → 一键下载/复制），可用步骤条或卡片。
- 平台展示：展示支持的平台（小红书、抖音、微信、B站、快手）图标或名称，增强信任感。
- Footer：版权、链接（如品牌语音、定价、联系）。

请输出：IA + 视觉规范 + React+Tailwind 代码，并说明响应式与首屏加载策略。
```

## 使用方式

- **在 Manus 网页/产品中**：把「通用项目上下文」+ 你要的某个 Prompt（如 Prompt 1）合并成一段话粘贴发送。
- **在 MCP 工具中**：若 `manus_generate_ui` 已接通，将合并后的文本传入 `prompt` 参数，`outputMode` 建议设为 `both`（先 spec 再 code）。

按需替换或删减「页面功能要点」即可生成其他页面（如结果详情、设置页等）。
