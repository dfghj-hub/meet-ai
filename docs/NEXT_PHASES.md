# 后续阶段目标

当前已完成：**阶段 0（项目初始化）、阶段 1（文字稿→纪要）、阶段 2（录音→纪要）**。  
以下为后续阶段的目标、产出与建议顺序，便于你决定「接下来先做哪一块」。

---

## 阶段 3：内测与反馈

| 项目 | 说明 |
|------|------|
| **目标** | 验证真实用户是否愿意用、是否愿意付费，收集改进点 |
| **产出** | 部署稳定的生产链接；10–20 人试用；一份反馈汇总（定价/功能优先级） |
| **何时做** | 阶段 2 跑通后即可做，不依赖阶段 4 |
| **你要做的** | 部署到 Vercel、把链接发给目标用户（产品/销售/自由职业）、用问卷或表格收集「会不会付钱、付多少、最缺什么」 |

---

## 阶段 4：登录 + 免费额度 + 付费入口

| 项目 | 说明 |
|------|------|
| **目标** | 区分匿名与登录用户；免费用户 3 次/月；用量用尽后展示升级 CTA；能收款（首版可手动开通） |
| **产出** | Supabase Auth + 用量表；免费/付费 tier 校验；定价页；至少一种支付渠道（爱发电/面包多/Lemon Squeezy） |
| **何时做** | 阶段 3 反馈显示「有人愿意付费」后再做，避免过早加登录影响传播 |
| **关键任务** | 见开发计划中 Task 4.1–4.3（Supabase、用量、定价页） |

---

## 阶段 5：打磨与推广

| 项目 | 说明 |
|------|------|
| **目标** | 提升结果页体验；通过内容/SEO 带来自然流量 |
| **产出** | 结果页体验优化（下载命名、复制反馈等）；至少 1 篇可对外发布的文章（如「如何高效做会议纪要」+ 产品链接）；落地页 SEO |
| **何时做** | 阶段 4 上线后，或与阶段 4 并行（先做静态定价页与文章） |

---

## 阶段 6：数据与迭代

| 项目 | 说明 |
|------|------|
| **目标** | 用数据决定下一版做「历史记录」还是「按人分段」、是否调价或加渠道 |
| **产出** | 基础指标（注册、用量、付费转化）；迭代决策文档（做啥、为啥不做啥） |
| **何时做** | 阶段 4–5 运行一段时间后，按周/月复盘 |

---

## 建议顺序

1. **先做阶段 3**：部署 + 内测 + 反馈，确认需求与付费意愿。  
2. **再做阶段 4**：登录与付费，把「愿意付的人」变成真实收入。  
3. **阶段 5、6**：随节奏做打磨、内容、数据与迭代。

若你希望「先收到第一笔钱再加功能」，可把阶段 4 提前（先做定价页 + 手动开通，再补登录与用量）。
