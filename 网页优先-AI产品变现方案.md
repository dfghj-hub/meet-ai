# 网页优先 + AI 能力：具体变现方案

> 路线：先做网页验证 → 赚到第一笔钱 → 再考虑 App（或小程序）。结合你的 AI 技术能力和充足时间，下面是可执行的方案。

---

## 一、整体路线图

```
第 1 阶段（约 4–8 周）  第 2 阶段（赚到钱之后）  第 3 阶段（可选）
─────────────────────────────────────────────────────────────
  做单一 AI 网页产品        稳定月收入 ¥2k+         做 App / 小程序
  ↓                        ↓                        ↓
  MVP 上线                 加功能、做订阅/套餐       PWA / 微信小程序 / 原生
  收第一笔钱               控制成本、优化转化       复用网页逻辑 + 壳
```

- **为什么先网页**：零上架费、随时改版、分享链接即用，适合验证需求。
- **何时做 App**：当网页月收入能覆盖 Apple 开发者年费（约 ¥688）且仍有结余、或用户明确要求「想要 App」时，再上架不迟。
- **App 形态可选**：PWA（加主屏）、微信小程序、或后期用 Capacitor/TWA 把网页包成 App，逻辑复用、成本低。

## 二、三个具体方案（任选其一先做）

每个方案都满足：**网页先行、强依赖 AI、你能独立完成、有清晰付费点**。

---

### 方案 A：AI 会议/语音 → 结构化笔记（会议助手）

**解决谁的问题**：开会多的人（产品、运营、销售、自由职业者），不想自己记要点、整理待办。

**产品形态（网页）**：
- 用户上传 **录音文件** 或 **会议链接/文字稿**（先支持上传文件即可）。
- 后端用 **语音转文字（Whisper 或国内 API）+ 大模型** 做：摘要、待办提取、按人/按话题分段、生成「会议纪要」可复制/下载。
- 免费：每月 2–3 次；付费：按次包月（如 30 次/月）或按分钟数。

**核心页面与流程**：
1. 首页：一句话说明 + 「上传录音/粘贴文字」+ 价格说明。
2. 上传页：拖拽或选择文件（mp3/wav/m4a），可选「只要摘要 / 要待办 / 要完整纪要」。
3. 结果页：展示摘要、待办列表、分段纪要，按钮「复制」「下载 txt/md」。
4. 定价页：免费额度用完 → 引导「解锁本月 30 次 ¥29」或「¥99 包年」。

**技术栈建议**：
- 前端：Next.js（App Router）+ Tailwind，部署在 Vercel。
- 后端：Next.js API Routes 或单独 Node 服务；语音转写用 OpenAI Whisper API 或阿里/腾讯/讯飞（国内更稳）；文本处理用 GPT-4o-mini 或国产大模型。
- 存储：用户文件临时存 Vercel Blob 或 S3 兼容（如 Cloudflare R2），处理完可定时删。
- 用户与计费：Supabase（Auth + 免费 PostgreSQL）记「用户 + 已用次数」；支付用 Lemon Squeezy / 爱发电 / 或国内支付（需主体）。

**定价参考**：
- 免费：3 次/月（或 30 分钟/月）。
- 个人版：¥29/月，30 次（或 300 分钟）。
- 年付：¥199/年，相当于约 ¥16.6/月。

**第一步（2 周内可跑通）**：
1. 做一个单页：上传一个音频 → 调用 Whisper 转文字 → 再调 GPT 输出「摘要 + 待办」→ 页面展示。  
2. 不登录、不付费，先发给 10 个目标用户用，问「你愿意为这个付多少钱？」。  
3. 有人愿意付再上「登录 + 次数限制 + 付费解锁」。

## 方案 A 深化：会议助手完整设计（Brainstorm 版）

以下是对方案 A 的扩展，便于直接按「产品 + 技术 + 运营」执行。

### 1. 用户画像与场景（更具体）

| 用户类型 | 典型场景 | 核心诉求 | 可接受的付费意愿 |
|----------|----------|----------|-------------------|
| 产品 / 项目 | 需求评审、站会、复盘 | 待办责任到人、决策留痕 | 中高（可报销） |
| 销售 / 客户成功 | 客户电话、内部过单 | 客户要点、下一步动作 | 中高 |
| 自由职业 / 咨询 | 客户会议、访谈 | 快速出纪要给客户 | 中（自费） |
| 自媒体 / 播客 | 录制访谈、对谈 | 出文字稿 + 金句/摘要 | 中 |
| 学生 / 研究者 | 组会、讲座录音 | 要点提取、复习用 | 低（偏免费） |

**产品定位建议**：优先服务「产品 / 销售 / 自由职业」—— 付费意愿明确、场景清晰。学生可保留免费额度做传播与口碑。

---

### 2. 输入方式（分阶段做）

| 输入方式 | MVP 是否做 | 说明 |
|----------|------------|------|
| 上传录音文件（mp3 / wav / m4a） | ✅ 必做 | 最通用，手机录音、会议录制均可 |
| 粘贴已有文字稿 | ✅ 必做 | 无转写成本，只做「摘要+待办」即可验证需求 |
| 上传视频（只抽音频） | 第二版 | 需抽轨，增加存储与处理时间 |
| 会议链接（如腾讯会议回放） | 后期 | 依赖平台开放能力，先不赌 |

**MVP 输入**：只做「上传音频」+「粘贴文字」两种，足以覆盖大部分场景并控制成本。

---

### 3. 输出内容（从简到丰）

**MVP 必出**（一页内展示）：
- **一段话摘要**（2–4 句）：会议主题、结论、下一步。
- **待办列表**：每条为「谁 / 做什么 / 可选截止时间」，可复制单条或全部。
- **完整纪要**：按时间或按话题分段的长文本，支持「复制全文」「下载 .txt / .md」。

**第二版可加**：
- **按人分段**（谁说了什么）：需说话人分离（diarization），讯飞/阿里部分能力支持，或先用「发言人 1/2/3」占位。
- **决策清单**：单独列出「已达成共识」的决策项。
- **一封跟进邮件草稿**：根据纪要和待办生成「会后的总结邮件」给参会人。

**可暂不做**：实时转写、与日历/飞书深度集成、多会议合并分析 —— 留给验证成功后再做。

---

### 4. 与竞品的差异（为什么用你）

| 竞品 / 类型 | 你的差异化 |
|-------------|------------|
| 飞书妙记 / 腾讯会议自带 | 单次、无需绑定企业；支持「只上传一个文件」即出结果，适合跨平台录音、外包、自由职业。 |
| Notion AI / 讯飞听见 | 更轻：不占工作流，网页打开即用；定价可做「按次/按包」更灵活。 |
| 通用 ChatGPT | 专门为「会议→纪要」调优的 prompt + 结构化输出，用户不用自己写提示词。 |

**一句话定位**：**「上传一段会议录音或文字稿，一分钟内得到摘要、待办和可复制的纪要」** —— 强调「单次、无绑定、出结果快」。

### 5. 页面与流程（细化到可画线框图）

1. **首页 / Landing**
   - 主标题 + 一句价值主张（见上）。
   - 两个入口：**上传录音** / **粘贴文字稿**。
   - 下方：输出示例（摘要 + 待办 mock 图）、价格简述（免费 3 次/月，付费 ¥29/月起）。
   - 页脚：隐私说明（录音仅用于处理、不存原文可强调）、联系/反馈。

2. **上传 / 输入页**
   - 上传：拖拽或点击，支持 mp3 / wav / m4a，限制单文件（如 50MB 或 60 分钟，先小一点便于控制成本）。
   - 粘贴：大文本框，占位符「粘贴会议记录或转写稿」。
   - 输出选项（MVP 可简化为一个）：勾选「要摘要」「要待办」「要完整纪要」（默认全选）。
   - 提交后：跳转「处理中」页，显示进度（转写中 → 分析中），超过 1 分钟可显示「长会议请稍候」。

3. **结果页**
   - 顶部：一句摘要（可单独复制）。
   - 区块 1：待办列表，每行「负责方 · 事项 · 截止」，按钮「复制全部待办」。
   - 区块 2：完整纪要（可折叠），按钮「复制全文」「下载 .txt」「下载 .md」。
   - 底部 CTA：若为免费用户且本月已用满，显示「解锁本月 30 次 ¥29」或「¥199 包年」。

4. **定价页**
   - 免费：3 次/月（或 30 分钟/月），仅摘要+待办+纪要，无历史记录。
   - 个人版：¥29/月，30 次（或 300 分钟/月），可选「历史记录保留 7 天」。
   - 年付：¥199/年（约 ¥16.6/月），同个人版权益，加「优先处理」文案即可。
   - 若用「按分钟」计费：例如免费 30 分钟/月，付费 300 分钟/月，需在后台严格按转写分钟数扣减，并考虑单次上限（如单次最多 2 小时）防滥用。

---

### 6. 成本与定价约束（重要）

- **语音转写**：Whisper 官方约 $0.6/分钟，成本高；国内阿里约 ¥0.058/分钟、讯飞类似。**建议**：主用国内 API（中文场景），或 Whisper 中转（$0.15/分钟级）并严格限制免费分钟数。
- **大模型**：GPT-4o-mini 或国产等价，单次会议纪要约 2k–8k tokens，成本可接受；按「次」或「分钟」封顶更易算。
- **存储**：用户音频处理完即删（或 24 小时内删），只存「用户 ID、用量、时间戳」；历史纪要若做，只存文本不存音频。

**定价建议**：免费 3 次/月（或 30 分钟/月）；付费 ¥29/月 30 次（或 300 分钟/月），保证单次会议成本 < ¥1，毛利空间才健康。

---

### 7. 技术要点（便于实现）

- **单文件时长/大小限制**：MVP 建议单次 ≤ 30 分钟 或 ≤ 25MB，避免首月被长会议拖垮成本。
- **队列与超时**：转写 + 大模型若超过 60 秒，可异步：提交后返回「处理中」，结果页轮询或 WebSocket；或发邮件/站内通知「结果已就绪」。
- **隐私与合规**：文案写清「录音仅用于生成纪要，不在服务器长期保存」；若做企业版再考虑合规条款。
- **计费维度二选一**：  
  - **按次**：简单，一次上传 = 扣 1 次；或按「音频+文字」分别算（粘贴文字 0.5 次）。  
  - **按分钟**：更公平，但需精确统计转写分钟数并展示在「用量页」。

### 8. 验证指标（上线后看什么）

| 指标 | 目标（参考） |
|------|----------------|
| 上传 → 成功出结果 | > 90% |
| 免费用户用完额度后的「升级点击」 | 追踪比例，若 > 20% 说明付费意愿存在 |
| 付费转化（免费 → 付费） | 首月 2–5% 即可继续迭代 |
| 单用户月均使用次数（付费用户） | 若 > 10 次/月，说明粘性够 |
| 分享/复访 | 通过 UTM 或「从哪来的」简单统计 |

### 方案 B：一文多发 / 多平台文案改写（内容助手）

**解决谁的问题**：做自媒体、公众号、小红书、知乎的人，同一件事要改写成不同风格、不同字数，费时。

**产品形态（网页）**：
- 用户粘贴 **一篇原文**（或上传文档），选择 **目标平台**（如：公众号、小红书、知乎、Twitter）和 **语气**（专业 / 轻松 / 卖货）。
- AI 输出：每个平台一版改写稿，带简单说明（如「小红书：强调标题和前 3 句」）。
- 免费：每日 1 次或 3 次；付费：每日 20 次 或 无限次。

**核心页面与流程**：
1. 首页：说明「一次输入，多平台改写」+ 平台图标 + 定价入口。
2. 编辑页：大文本框粘贴原文；多选「目标平台」；选择语气/风格。
3. 结果页：按平台 Tab 或折叠展示各版本文案，每段可复制。
4. 用量与付费：顶部显示「今日已用 x/3」，用尽后弹窗「升级 ¥19/月 每日 20 次」。

**技术栈建议**：
- 与方案 A 类似：Next.js + Vercel，Supabase 做用户与用量。
- 只调大模型 API（无语音），成本更低；用 system prompt 固定「平台风格规则」，便于稳定质量。

**定价参考**：
- 免费：3 次/天。
- 基础版：¥19/月，20 次/天。
- 进阶版：¥49/月，不限次（可设单次上限防滥用）。

**第一步（1–2 周）**：
1. 单页：一个输入框 + 选 2 个平台（如公众号 + 小红书）→ 调 API 出 2 段改写 → 展示。  
2. 找 5–10 个做内容的朋友试用，看是否愿意为「省时间」付费。  
3. 再加登录、次数、支付。

### 方案 C：AI 小工具集（工具箱 + 1–2 个付费项）

**解决谁的问题**：不想在多个网站之间切换的人，希望「一个站搞定几件小事」；其中 1–2 个工具足够疼，就值得付费。

**产品形态（网页）**：
- 一个导航页，列出 5–8 个 **AI 小工具**，例如：
  - 长文章/链接 → 摘要（可带「3 条要点」）
  - 商品描述/标题 → 改写/扩写（电商用）
  - 简历/JD 文本 → 匹配度分析 + 改写建议
  - 小红书/公众号标题生成（给关键词出 10 个标题）
- 大部分免费但带次数限制；**1–2 个**设为「高级」：要么单次付费（如 ¥0.5/次），要么包月（如 ¥15/月 畅用）。

**技术栈建议**：
- 同上：Next.js + Vercel + Supabase；每个工具一个路由（如 `/tools/summary`、`/tools/rewrite`）。
- 工具内部：表单 → API Route → 调大模型 → 返回结果。先不接支付，用「免费 N 次/天」验证哪个工具被用得最多，再对那个做付费。

**定价参考**：
- 全站免费：每工具 5 次/天。
- 高级：¥15/月，所有工具 50 次/天；或单工具「¥0.3/次」按量扣。

**第一步（2–3 周）**：
1. 先做 3 个工具：摘要、标题生成、简历/JD 匹配。  
2. 上线到 Vercel，挂在一个好记的域名下，在知乎/小红书发「我做了个免费的 XX 工具箱」。  
3. 看后台或埋点：哪个工具使用最多、用户是否返回；再给该工具加「高级版」付费。

## 四、网页 → App 的后续路径（赚到钱再做）

1. **PWA**  
   - 在网页里加 `manifest.json` + Service Worker，用户「添加到主屏」即可当 App 用。  
   - 适合「偶尔打开、分享链接」的场景，零上架成本。

2. **微信小程序**  
   - 若用户主要在微信里分享/使用，可用同一套业务逻辑（API 不变），用小程序前端调你的后端。  
   - 无 Apple 开发者费，审核相对可预期。

3. **Capacitor / TWA 包一层**  
   - 用 Capacitor 把现有网页包成 iOS/Android App，或用 TWA（Trusted Web Activity）上架 Google Play。  
   - 逻辑全在网页，维护一份代码即可；等收入稳定再交 Apple 年费上架 App Store。

4. **真正原生 App**  
   - 当网页 + 小程序月收入能稳定覆盖 ¥688/年且你希望「在 App Store 有个入口」时，再考虑 React Native / Swift 等重写或重做客户端。

## 五、执行顺序建议（按周）

- **第 1–2 周**：从 A/B/C 里选一个你最顺手或最想做的；只做「单次使用、无登录」的 MVP（例如方案 A 只做「上传 → 出纪要」）。  
- **第 3 周**：内测 + 小范围分享（10–20 人），收集「会不会付钱、付多少」的反馈。  
- **第 4 周**：加上「登录 + 免费次数 + 付费入口」（可先接爱发电/面包多手动开通，再考虑自动支付）。  
- **第 5–8 周**：根据反馈加 1–2 个核心功能，并做简单 SEO/内容（写一两篇「如何整理会议纪要」类文章挂上产品链接）。  
- **持续**：每周看「注册数、付费数、哪个功能用得最多」，再决定是否做第二个产品或加 App/小程序。

---

## 六、总结

- **先网页**：零上架费、迭代快、方便分享，适合验证。  
- **用 AI**：会议转纪要、一文多发、小工具集都能发挥你的 AI 能力，且技术栈统一。  
- **具体**：从「会议助手 / 多平台改写 / 工具箱」里选一个，按上面「第一步」先做出可用的单点流程，再加登录与付费。  
- **再做 App**：等网页端有稳定小收入后再考虑 PWA → 小程序 → 包壳上架，避免先付 Apple 年费却还没验证需求。

**详细开发计划（含多 Agent 分工）：** 已写入 `docs/plans/2025-02-22-meeting-assistant-implementation-plan.md`，包含阶段 0–6 的详细任务、验收标准与可并行执行的任务分配；确认可行后可直接按该计划执行或派发多 agent 并行开发。
